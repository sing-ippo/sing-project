## Правила (без исключений)

1. **Работаешь в своей ветке `team-...`**. В `main` руками не пушим — будет включён branch protection
2. **Финальная сдача = PR в main**. Ревьюит другой студент группы (не Сергей Романович), потом мерджит
3. **README.md в твоей папке обязателен** и должен содержать:
   - 2–3 предложения «что это делает»
   - Какие системные программы поставить (`ffmpeg`, `docker` и т.д.)
   - Какие python-зависимости (`pip install -r requirements.txt`)
   - Какие переменные окружения нужны (см. `.env.example`)
   - Команду запуска
   - Пример проверки что работает (`curl` или скриншот)
4. **`.env.example` в твоей папке** — шаблон переменных без значений. Реальный `.env` коммитить НЕ нужно, его в `.gitignore`
5. **Smoke test** — другой студент клонирует репо, читает твой README, запускает у себя. Не запустилось — не сдано
