# СИНГ ИиППО — Единые форматы данных

Этот документ фиксирует **все** форматы ввода-вывода для обеих команд. Любой модуль, скрипт или бот должен работать именно с этими структурами. Кодировка всех файлов — **UTF-8**.

---

## Команда «Ассистент студента»

### 1. `faq.json` — база знаний бота

Единый формат для всех источников FAQ: ручные записи, данные из чатов, лекционный контент.

```json
[
  {
    "id": 1,
    "question": "Где посмотреть расписание?",
    "keywords": ["расписание", "пары", "занятия"],
    "answer": "Расписание можно найти на mirea.ru в разделе «Студентам»",
    "category": "навигация",
    "source": "manual"
  }
]
```

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `id` | `int` | да | Уникальный ID. Диапазоны: 1–999 — бытовой FAQ, 1000–1999 — лекции, 2000+ — из чатов |
| `question` | `str` | да | Формулировка вопроса |
| `keywords` | `list[str]` | да | Ключевые слова для поиска (строчные, без знаков препинания) |
| `answer` | `str` | да | Текст ответа |
| `category` | `str` | да | Категория: `навигация`, `экзамены`, `преподаватели`, `учебный_процесс`, `ит_системы`, `бюрократия`, `общежитие`, `лекции` |
| `source` | `str` | да | Откуда запись: `manual`, `chat_экзамены_мирэа`, `chat_поступление_иит`, `chat_подслушано_мирэа`, `lecture_<id>` |

---

### 2. `student_questions.json` — сырые вопросы из чатов

Промежуточный формат для данных, собранных из Telegram-чатов. Используется для аналитики и последующего преобразования в `faq.json`.

```json
[
  {
    "id": 1,
    "question": "Когда будет пересдача по матану?",
    "category": "экзамены",
    "source": "chat_экзамены_мирэа",
    "frequency": 12
  }
]
```

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `id` | `int` | да | Порядковый номер |
| `question` | `str` | да | Текст вопроса (цитата или обобщение) |
| `category` | `str` | да | Одна из категорий (см. `faq.json`) |
| `source` | `str` | да | Источник: `chat_экзамены_мирэа`, `chat_поступление_иит`, `chat_подслушано_мирэа` |
| `frequency` | `int` | да | Сколько раз встретился похожий вопрос |

---

### 3. `search_result` — формат результата поиска (Python-объект, не файл)

Возвращается функцией `search()`. Не сохраняется в файл — используется внутри бота.

```json
{"id": 1, "question": "Где расписание?", "answer": "Расписание на mirea.ru", "score": 8}
```

| Поле | Тип | Описание |
|---|---|---|
| `id` | `int` | ID записи из `faq.json` |
| `question` | `str` | Вопрос из FAQ |
| `answer` | `str` | Ответ из FAQ |
| `score` | `int` | Оценка релевантности (exact=4, partial=3, prefix=2, суммируется по словам) |

---

### 4. `requests.jsonl` — лог запросов бота

Каждая строка — один запрос пользователя. JSON Lines.

```json
{"timestamp": "2026-03-18T14:30:00", "query": "когда стипендия", "answer": "Стипендия приходит...", "matched": true, "helpful": null}
```

| Поле | Тип | Описание |
|---|---|---|
| `timestamp` | `str` (ISO 8601) | Время запроса |
| `query` | `str` | Текст вопроса пользователя |
| `answer` | `str` | Ответ бота (или «Не знаю...») |
| `matched` | `bool` | Нашёлся ли ответ в FAQ |
| `helpful` | `bool \| null` | Оценка пользователя: помог ли ответ (null если отзыв не оставлен) |

---

### 5. `suggestions.jsonl` — предложения от пользователей

Каждая строка — одно предложение. JSON Lines.

```json
{"timestamp": "2026-03-18T15:00:00", "user": "Иван", "type": "new_question", "question": "Как подать документы?", "answer": null}
```

| Поле | Тип | Описание |
|---|---|---|
| `timestamp` | `str` (ISO 8601) | Время предложения |
| `user` | `str` | Имя пользователя |
| `type` | `str` | Тип: `new_question`, `new_answer`, `question_with_answer` |
| `question` | `str` | Текст вопроса |
| `answer` | `str \| null` | Текст ответа (null, если тип = `new_question`) |

## Команда «Ассистент преподавателя»

### 6. `chunks.jsonl` — нарезанный документ

Каждая строка — один чанк. JSON Lines.

```json
{"chunk_id": 1, "source": "методичка.pdf", "page": 1, "text": "Текст фрагмента...", "word_count": 387, "overlap_prev": false}
```

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `chunk_id` | `int` | да | Порядковый номер чанка (начиная с 1) |
| `source` | `str` | да | Имя исходного файла |
| `page` | `int \| null` | да | Номер страницы (null для .txt и .docx) |
| `text` | `str` | да | Текст чанка |
| `word_count` | `int` | да | Количество слов в чанке |
| `overlap_prev` | `bool` | да | Начало чанка содержит перекрытие с предыдущим |

**Параметры по умолчанию:** chunk_size=400 слов, overlap=50 слов.

---

### 7. `quiz.json` — сгенерированный тест

```json
[
  {
    "id": 1,
    "question": "Что называется динамической типизацией?",
    "options": ["Вариант A", "Вариант B", "Вариант C", "Вариант D"],
    "correct": 1,
    "explanation": "Интерпретатор сам определяет тип переменной при присвоении",
    "source_chunk_id": 3
  }
]
```

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `id` | `int` | да | Порядковый номер вопроса |
| `question` | `str` | да | Текст вопроса |
| `options` | `list[str]` | да | 4 варианта ответа |
| `correct` | `int` | да | Индекс правильного ответа (0–3) |
| `explanation` | `str` | да | Объяснение правильного ответа |
| `source_chunk_id` | `int \| null` | да | Ссылка на `chunk_id` из `chunks.jsonl` (null, если текст не из чанка) |

---

### 8. `formulas.jsonl` — извлечённые формулы

Каждая строка — одна формула. JSON Lines.

```json
{"formula_id": 1, "page": 5, "latex": "x^2 + 3y", "context": "Рассмотрим выражение...", "method": "pix2tex", "confidence": 0.85}
```

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `formula_id` | `int` | да | Порядковый номер формулы |
| `page` | `int` | да | Номер страницы в PDF |
| `latex` | `str \| null` | да | Формула в LaTeX (null, если не удалось извлечь) |
| `context` | `str` | да | Текст вокруг формулы (1–2 предложения) |
| `method` | `str` | да | Метод извлечения: `pix2tex`, `nougat`, `pdfplumber` |
| `confidence` | `float \| null` | да | Уверенность от 0.0 до 1.0 (null, если метод не даёт оценку) |

## Пайплайн преподавателя (CLI)

```
python pipeline.py --input <файл> --output <папка>
```

Результат в `<папка>/`:

- `chunks.jsonl` — нарезанный документ
- `quiz.json` — сгенерированный тест

## Правила именования source

| Источник | Значение `source` |
|---|---|
| Ручная запись | `manual` |
| Чат «Экзамены МИРЭА» | `chat_экзамены_мирэа` |
| Чат «Поступление ИИТ 2025» | `chat_поступление_иит` |
| Чат «Подслушано МИРЭА» | `chat_подслушано_мирэа` |
| Лекция (по ID из lectures.json) | `lecture_<id>` (например, `lecture_1`) |
