# SING Knowledge Base

Проект собирает единую базу знаний для студенческого ассистента SING из нескольких источников: ручных FAQ-записей, лекций, вопросов из студенческих чатов и материалов Дня открытых дверей.

Итоговый результат — файл `faq.json`. Его можно использовать для поиска ответов, Telegram-бота, квизов и дальнейшего перехода на базу данных.

---

## Требования и установка

Проект рассчитан на Python 3.11+.

### 1. Создать виртуальное окружение

Windows PowerShell:

```bash
python -m venv .venv
.venv\Scripts\activate
```

Linux / macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Основные зависимости

```text
pymorphy3>=2.0.0
nltk>=3.8.1
python-dotenv>=1.0.0
```

| Пакет | Для чего используется |
|---|---|
| `pymorphy3` | Морфологическая нормализация русских слов: `деканате → деканат`, `экзамены → экзамен` |
| `nltk` | Стоп-слова и дополнительная текстовая обработка в `build_faq.py` |
| `python-dotenv` | Необязательная зависимость для будущей работы с `.env`, токенами и настройками |

Важно: `convert_chats.py` и `convert_lectures.py` не должны зависеть от `nltk`. Финальная нормализация ключевых слов выполняется в `build_faq.py`.

Если при запуске появляется ошибка про `stopwords`, один раз выполните:

```bash
python -c "import nltk; nltk.download('stopwords')"
```

## 1. Назначение проекта

Проект нужен для подготовки базы знаний студенческого помощника.

Основные возможности:

- сбор FAQ из нескольких источников;
- очистка и фильтрация вопросов из чатов;
- генерация FAQ по лекциям;
- добавление FAQ по Дню открытых дверей;
- нормализация ключевых слов через морфологию;
- удаление дублей;
- валидация итогового формата;
- генерация отчёта о сборке;
- подготовка данных для Telegram-бота с квизами.

## 4. Структура входных JSON-файлов

### 4.1. `faq_manual.json`

Ручные проверенные FAQ-записи.

Это самый надёжный источник базы: расписание, общежитие, учебный отдел, пропуск, ВУЦ, стипендия и другие часто задаваемые вопросы.

Структура:

```json
[
  {
    "id": 1,
    "question": "Где расписание?",
    "keywords": [
      "расписание",
      "пара",
      "занятие"
    ],
    "answer": "Расписание можно найти на mirea.ru в разделе «Студентам».",
    "category": "навигация",
    "source": "manual"
  }
]
```

Поля:

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `id` | `int` | Да | Уникальный ID записи |
| `question` | `str` | Да | Текст вопроса |
| `keywords` | `list[str]` | Да | Ключевые слова для поиска |
| `answer` | `str` | Да | Готовый ответ |
| `category` | `str` | Да | Категория вопроса |
| `source` | `str` | Да | Источник записи, для ручных записей — `manual` |

`faq_manual.json` не требует предварительной конвертации и используется напрямую в `build_faq.py`.

---

### 4.2. `lectures.json`

Исходные материалы лекций.

Файл содержит список лекций. Он используется в `convert_lectures.py` вместе с `lecture_faq.json`.  
По полю `id` лекция связывается с вопросами из `lecture_faq.json`.

Структура:

```json
[
  {
    "id": 1,
    "title": "Введение в Python. Типы данных",
    "subject": "Программирование",
    "source": "Habr",
    "key_points": [
      "Python — интерпретируемый язык",
      "Основные типы: bool, int, float, complex, str, tuple, list, dict, set",
      "Динамическая типизация — тип определяется автоматически"
    ],
    "video_url": null
  }
]
```

Поля:

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `id` | `int` | Да | Номер лекции. Связывается с `lecture_id` из `lecture_faq.json` |
| `title` | `str` | Да | Название лекции |
| `subject` | `str` | Да | Учебный предмет или направление |
| `source` | `str` | Да | Источник лекционного материала |
| `key_points` | `list[str]` | Да | Основные тезисы лекции |
| `video_url` | `str` или `null` | Нет | Ссылка на видео, если она есть |

Пример связи с `lecture_faq.json`:

```text
lectures.json:     "id": 1
lecture_faq.json:  "lecture_id": 1
```

---

### 4.3. `lecture_faq.json`

Подготовленные вопросы и ответы по лекциям.

Этот файл содержит заготовки вопросов. На его основе `convert_lectures.py` создаёт `faq_lectures.json`.

Структура:

```json
[
  {
    "lecture_id": 1,
    "question": "Что такое динамическая типизация?",
    "answer": "Тип переменной определяется автоматически при присвоении — не нужно объявлять его заранее"
  }
]
```

Поля:

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `lecture_id` | `int` | Да | ID лекции из `lectures.json` |
| `question` | `str` | Да | Вопрос по лекции |
| `answer` | `str` | Да | Правильный ответ |

После конвертации запись получает общий FAQ-формат:

```json
[
  {
  "id": 1000,
  "question": "Что такое динамическая типизация?",
  "keywords": [
    "python",
    "динамическая",
    "типизация"
  ],
  "answer": "Тип переменной определяется автоматически при присвоении — не нужно объявлять его заранее",
  "category": "лекции",
  "source": "lecture_1"
  }
]
```

В итоговой FAQ-базе лекционные записи имеют:

```text
category = "лекции"
source = "lecture_<номер лекции>"
```

### 4.4. `student_questions.json`

Сырые вопросы и сообщения из студенческих чатов.

Этот файл используется в `convert_chats.py`. Сообщения проходят очистку, фильтрацию и группировку. Полезные вопросы попадают в `faq_chats.json`, а отклонённые сообщения — в `rejected_chats.json`.

Основная структура:

```json
[
  {
    "id": 1,
    "question": "привет хочу перевестись с лазерки на дизайн получится или нет?",
    "answer": "None",
    "keywords": [
      "получаться"
    ],
    "category": "учебный_процесс",
    "source": "chat_подслушано_мирэа",
    "frequency": 1
  },
  {
    "id": 2,
    "question": "сколько реально на экзаменах в первом семестре баллов набрать?",
    "answer": "None",
    "keywords": [
      "балл",
      "экзамен"
    ],
    "category": "экзамены",
    "source": "chat_подслушано_мирэа",
    "frequency": 1
  }
]
```

Поля:

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `id` | `int` | Да | Уникальный ID исходного вопроса из чата |
| `question` | `str` | Да | Текст вопроса или сообщения из студенческого чата |
| `answer` | `str` / `null` | Нет | Ответ на вопрос. На этапе сырых чатовых данных обычно пустой, `null` или `"None"` |
| `keywords` | `list[str]` | Нет | Ключевые слова, если они уже были заранее выделены |
| `category` | `str` | Нет | Категория вопроса, если она уже была определена до обработки |
| `source` | `str` | Нет | Источник сообщения, например `chat_подслушано_мирэа` |
| `frequency` | `int` | Нет | Количество повторений похожего вопроса или частота появления |

Если `source` отсутствует, используется значение по умолчанию:

```text
chat_подслушано_мирэа
```

После обработки создаётся запись:

```json
  [
{
  "id": 2000,
  "question": "Как закрыть долг по экзамену?",
  "keywords": [
    "закрыть",
    "долг",
    "экзамен"
  ],
  "answer": "Ответ будет добавлен после модерации.",
  "category": "экзамены",
  "source": "chat_подслушано_мирэа"
}
  ]
```

### 4.5. `dod_raw.json`

Черновые данные по Дню открытых дверей.

Этот файл содержит вопросы абитуриентов. В нём не нужно вручную указывать `id`, `category` и `source`, потому что эти поля добавляет `parse_dod.py`.

Структура:

```json
[
  {
    "question": "Какие документы нужны для поступления?",
    "keywords": [
      "документы",
      "поступление",
      "приемная комиссия"
    ],
    "answer": "Для поступления обычно нужны паспорт, документ об образовании, заявление, СНИЛС и другие документы, указанные в правилах приёма."
  }
]
```

Поля:

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `question` | `str` | Да | Вопрос по поступлению |
| `keywords` | `list[str]` | Да | Ключевые слова |
| `answer` | `str` | Да | Ответ |

Все записи из `dod_raw.json` относятся к категории `поступление`.

### 4.6. `dod_faq.json`

Готовый FAQ-файл по Дню открытых дверей.

Он создаётся после запуска:

```bash
python parse_dod.py
```

`parse_dod.py` читает `dod_raw.json`, добавляет служебные поля и сохраняет результат в `sources/dod_faq.json`.

Структура:

```json
[
  {
    "id": 30000,
    "question": "Какие документы нужны для поступления?",
    "keywords": [
      "документы",
      "поступление",
      "приемная комиссия"
    ],
    "answer": "Для поступления обычно нужны паспорт, документ об образовании, заявление, СНИЛС и другие документы, указанные в правилах приёма.",
    "category": "поступление",
    "source": "dod"
  }
]
```

Поля:

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `id` | `int` | Да | Уникальный ID записи |
| `question` | `str` | Да | Вопрос |
| `keywords` | `list[str]` | Да | Ключевые слова |
| `answer` | `str` | Да | Ответ |
| `category` | `str` | Да | Всегда `поступление` |
| `source` | `str` | Да | Всегда `dod` |

## 6. Что делает каждый Python-файл

### `convert_lectures.py`

Преобразует лекционные материалы в FAQ-записи.

Основные задачи:

- загрузить `sources/lectures.json`;
- загрузить `sources/lecture_faq.json`;
- связать вопросы с лекциями по `lecture_id`;
- собрать ключевые слова из вопроса, ответа и тезисов лекции;
- сохранить результат в `faq_lectures.json`.

---

### `convert_chats.py`

Преобразует сообщения из чатов в FAQ-кандидаты.

Основные задачи:

- загрузить `sources/student_questions.json`;
- очистить текст сообщений;
- отфильтровать мусор, флуд, рекламу и неучебные сообщения;
- определить категорию вопроса;
- собрать ключевые слова;
- сгруппировать одинаковые вопросы;
- сохранить принятые вопросы в `faq_chats.json`;
- сохранить отклонённые вопросы в `rejected_chats.json`;
- вывести статистику принятых и отклонённых сообщений.

Пример статистики:

```text
Загружено сообщений: 8849
Принято вопросов: 1200
Отклонено сообщений: 7649
После группировки: 980
Сохранено FAQ-записей: 980
```

---

### `parse_dod.py`

Создаёт готовые FAQ-записи по Дню открытых дверей.

Основные задачи:

- загрузить `sources/dod_raw.json`;
- добавить `id`, `category`, `source`;
- сохранить результат в `sources/dod_faq.json`.

---

### `build_faq.py`

Главный файл сборки базы знаний.

Основные этапы:

1. загрузить все источники;
2. объединить записи;
3. нормализовать ключевые слова;
4. удалить дубли;
5. выполнить валидацию;
6. сохранить итоговый `faq.json`;
7. при ошибках сохранить `faq_with_errors.json`;
8. записать ошибки в `errors.log`;
9. создать `report.txt`.

Примеры нормализации:

```text
деканата → деканат
деканате → деканат
экзамены → экзамен
поступления → поступление
воёнок → военный
общага → общежитие
препод → преподаватель
```

---

### `dedup.py`

Удаляет дублирующиеся FAQ-записи.

Основные задачи:

- найти повторяющиеся или похожие записи;
- не удалять разные лекции как дубли;
- оставить более полную запись;
- вернуть количество удалённых дублей.

---

### `validate.py`

Проверяет корректность FAQ-записей.

Проверяются:

- наличие обязательных полей;
- типы данных;
- пустые значения;
- уникальность `id`;
- корректность `category`;
- корректность `source`;
- соответствие диапазонов ID источникам.

## 7. Порядок запуска

## 9. Формат итогового `faq.json`

Итоговый файл `faq.json` — это список объектов единого формата:

```json
[
  {
    "id": 1,
    "question": "Где расписание?",
    "keywords": [
      "расписание",
      "пара",
      "занятие"
    ],
    "answer": "Расписание можно найти на mirea.ru в разделе «Студентам».",
    "category": "навигация",
    "source": "manual"
  }
]
```

Поля итоговой записи:

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `id` | `int` | Да | Уникальный ID записи |
| `question` | `str` | Да | Текст вопроса |
| `keywords` | `list[str]` | Да | Нормализованные ключевые слова |
| `answer` | `str` | Да | Ответ |
| `category` | `str` | Да | Категория вопроса |
| `source` | `str` | Да | Источник записи |

Допустимые категории:

```text
навигация
учебный_процесс
экзамены
общежитие
ит_системы
бюрократия
преподаватели
лекции
поступление
```

Примеры источников:

```text
manual
lecture_1
lecture_2
chat_подслушано_мирэа
chat_экзамены_мирэа
chat_поступление_иит
dod
```

Рекомендуемые диапазоны ID:

```text
1–999        manual
1000–1999    lectures
2000+        chats
30000+       dod
```

DOD лучше держать в большом диапазоне, чтобы ID не пересекались с чатами.

## 10. Формат `rejected_chats.json`

`rejected_chats.json` содержит сообщения, которые не прошли фильтрацию.

Пример:

```json
[
  {
    "question": "спасибо",
    "source": "chat_подслушано_мирэа",
    "reason": "stop_message"
  }
]
```

Поля:

| Поле | Тип | Описание |
|---|---|---|
| `question` | `str` | Отклонённое сообщение |
| `source` | `str` | Источник сообщения |
| `reason` | `str` | Причина отклонения |

Возможные причины отклонения:

```text
empty
stop_message
too_short
too_long
no_letters
bad_phrase
bad_words
spaced_bad_words
compact_bad_words
no_question_meaning
no_study_marker
```

## 11. Формат `report.txt`

`report.txt` содержит текстовый отчёт о сборке.

В отчёте указывается:

```text
сколько записей загружено из manual
сколько записей загружено из lectures
сколько записей загружено из chats
сколько записей загружено из dod
сколько записей было до дедупликации
сколько дублей удалено
сколько записей осталось после дедупликации
сколько ошибок валидации найдено
куда сохранён итоговый faq.json
```

## 12. Формат квизов для Telegram-бота

Для квизов используется отдельный формат, например в файле `lectures_faq_quizs.json`.

```json
[
  {
    "lecture_id": 1,
    "question": "Что такое динамическая типизация?",
    "category": "Python",
    "topic": "Типы данных",
    "answer": "Тип переменной определяется автоматически при присвоении — не нужно объявлять его заранее"
  }
]
```

Поля:

| Поле | Тип | Описание |
|---|---|---|
| `lecture_id` | `int` | Номер лекции |
| `category` | `str` | Крупный раздел: Python, Git, Математика, SQL |
| `topic` | `str` | Конкретная тема |
| `question` | `str` | Вопрос |
| `answer` | `str` | Правильный ответ |

Логика бота:

```text
/start
↓
выбор category
↓
выбор topic
↓
вопросы из выбранной темы
↓
ответ пользователя
↓
показ правильного ответа
```

Примеры категорий:

```text
Python
Git
Математика
SQL
```

Примеры тем:

```text
Типы данных
Условия
Циклы
Основы Git
Удалённые репозитории
Коммиты и история
Ветки и слияние
Пределы последовательности
Матрицы
Реляционные базы данных
```

## 13. Пример полного сценария работы

1. Подготовить исходные файлы в `sources/`.
2. Запустить конвертеры.
3. Собрать итоговую базу.
4. Проверить валидацию.
5. Использовать `faq.json` в поиске или боте.
6. Использовать `lectures_faq_quizs.json` для квизов.

Команды:

```bash
python parse_dod.py
python convert_lectures.py
python convert_chats.py
python build_faq.py
python validate.py faq.json
```

## 14. Итог

Проект представляет собой пайплайн подготовки базы знаний:

```text
источники данных
↓
конвертация
↓
очистка
↓
нормализация
↓
дедупликация
↓
валидация
↓
faq.json
```

Такой подход позволяет постепенно развивать проект: сначала использовать JSON-файлы, затем перейти на базу данных, добавить Telegram-бота, квизы, модерацию и аналитику пользовательских запросов.
