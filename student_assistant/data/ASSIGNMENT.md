### Что нового в Sprint 3

Сейчас данные разбросаны: FAQ Льва — отдельно, лекции Вити — отдельно, чаты Саши — отдельно. Форматы немного отличаются, есть дубли, нет проверки качества.

**Новая функциональность:** один скрипт, который берёт ВСЕ источники и выдаёт чистый, валидный `faq.json` + отчёт о качестве данных.

1. **Конвертеры** — каждый источник → единый формат faq.json (поля, ID-диапазоны, source по FORMATS.md).

2. **Дедупликация** — если два вопроса совпадают на ≥ 80% слов, оставляем один (с бОльшим frequency или более полным ответом).

3. **Валидация** — проверка каждой записи: все обязательные поля есть, ID в правильном диапазоне, keywords не пустые, category из разрешённого списка.

4. **Отчёт** — статистика: сколько записей из каждого источника, сколько дублей удалено, сколько ошибок найдено и исправлено.

### Кто что делает

**Витя:**
- Доработать `convert_lectures.py` из Sprint 2:
  - Вход: `lectures.json` (каталог лекций)
  - Выход: список dict в формате faq.json с ID 1000+, source = `lecture_<id>`
  - Keywords: слова ≥ 3 символов + фильтрация стоп-слов (при, для, как, что, это)
- Написать `validate.py`:
  - Функция `validate_faq(entries: list[dict]) -> tuple[list[dict], list[str]]` — возвращает (валидные записи, список ошибок)
  - Проверяет: id (int), question (непустой str), keywords (непустой list), answer (непустой str), category (из списка), source (формат по FORMATS.md)
  - CLI: `python validate.py faq.json` → печатает ошибки или «OK, 0 ошибок»

**Александр:**
- Написать `convert_chats.py`:
  - Вход: `student_questions.json` (его формат из Sprint 2)
  - Выход: список dict в формате faq.json с ID 2000+, source = `chat_*`
  - Для каждого вопроса: генерация keywords из текста, назначение category
  - Пропускать вопросы с frequency < 3 (слишком редкие)
- Написать `dedup.py`:
  - Функция `deduplicate(entries: list[dict]) -> tuple[list[dict], int]` — возвращает (уникальные записи, количество удалённых дублей)
  - Два вопроса считаются дублями если ≥ 80% слов совпадают (после приведения к нижнему регистру)

**Вместе:**
- `build_faq.py` — главный скрипт, который вызывает конвертеры → merge → dedup → validate → сохраняет результат

1. `python build_faq.py --output faq.json` выполняется без ошибок
2. Открываю `report.txt` — вижу статистику по источникам и дублям
3. `python validate.py faq.json` → «OK, 0 ошибок»
4. В `faq.json`:
   - ≥ 50 записей
   - ID из трёх диапазонов (1–999, 1000–1999, 2000+)
   - Нет дублирующихся ID
   - Все поля заполнены по FORMATS.md
