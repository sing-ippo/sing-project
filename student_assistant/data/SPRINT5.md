## Витя — `build_faq.py` и сборка

### 1) `requirements.txt`

Пропиши все используемые библиотеки. Должно быть как минимум:
```
pymorphy3
nltk
```
Если используешь ещё (pdfplumber, python-docx и т.д.) — добавь.

### 2) Автоскачивание стоп-слов NLTK — ПРОВЕРКА

У тебя в `build_faq.py` уже есть в функции `build_stop_keywords()`:
```python
try:
    return set(stopwords.words("russian")) | CUSTOM_STOP_KEYWORDS
except LookupError:
    nltk.download("stopwords", quiet=True)
    return set(stopwords.words("russian")) | CUSTOM_STOP_KEYWORDS
```

Это правильно. **Просто проверь что этот блок остался на месте** — он гарантирует что у нового пользователя скрипт сам скачает стоп-слова при первом запуске.

### 3) Минимум 80 записей в итоговом `faq.json`

Запусти `python build_faq.py`, открой результат, посчитай длину массива. Если меньше 80 — допиши вручную в `sources/faq_manual.json` записей про типовые студенческие вопросы (расписание, дедлайны, пересдачи, как добраться до корпуса, где взять справку).

### 4) README


## Саша — чистка и улучшение данных

В `sources/student_questions.json` уже лежит **8995 записей**, но качество низкое: много мусорных сообщений типа «привет хочу перевестись с лазерки на дизайн», `frequency = 1` у всех (не агрегировано). Твоя задача — превратить эту свалку в полезные данные.

### 1) Отфильтруй мусор

Открой `student_questions.json`, удали записи которые:
- Не являются вопросами (нет смысловой просьбы информации)
- Содержат мат/оскорбления
- Слишком короткие (<5 слов) или бессмысленные
- Дубликаты по смыслу

Цель — оставить **100-200 качественных вопросов** на типовые студенческие темы. Можно написать скрипт `clean_questions.py` который делает фильтрацию.

### 2) Сделай нормальный `frequency`

Сейчас у всех записей `frequency = 1`. Сгруппируй похожие вопросы (например все вариации «когда пересдача» → одна запись с `frequency = 12`). Можно вручную через сортировку и группировку, или через скрипт.

### 3) Проверь интеграцию

Запусти `python build_faq.py` (из папки `student_assistant/data/`) и убедись что итоговый `faq.json` собирается без ошибок и содержит твои очищенные записи.

### 4) Сделай минимум 3 коммита в ветку `team-vitya-sasha`

Требование группы — каждый участник пары делает свой видимый вклад. В Sprint 4 коммитов от тебя не было, в этом спринте — обязательно.

## Как проверяется

1. Другой студент клонирует репо
2. `pip install -r requirements.txt`
3. `python build_faq.py` — стартует без ошибок (включая первый запуск с автоскачиванием NLTK)
4. `faq.json` создаётся, содержит ≥80 записей из трёх источников
5. Валидация (`validate.py`) проходит без ошибок
6. `student_questions.json` отфильтрован: 100-200 качественных записей, не 8995 мусорных
