### Что нового в Sprint 3

Сейчас бот отвечает на вопрос и всё — пользователь уходит. Не понятно: помог ответ или нет? Какие вопросы бот не покрывает?

**Новая функциональность:** бот собирает обратную связь и учится на пробелах.

1. **Кнопки «Помогло? Да / Нет»** — после каждого ответа бот спрашивает, был ли ответ полезен. Результат записывается в `requests.jsonl` (новое поле `helpful: true/false/null`).

2. **Сбор вопросов без ответа** — если поиск вернул score < min_score, бот пишет: «Не нашёл ответа. Хочешь оставить вопрос? Мы добавим его позже.» Если пользователь соглашается → запись в `suggestions.jsonl` (формат из FORMATS.md).

3. **Команда `/gaps`** — выводит список вопросов, на которые бот не смог ответить, сгруппированных по частоте. Это кандидаты для новых записей в FAQ.

### Кто что делает

**Артём:**

  - Новый метод `search_with_confidence(query, faq_data, top_n)` → возвращает результаты + флаг `confident: bool` (true если лучший score >= 4, false если все результаты < 4)

  - Функция `get_gaps(path, top_n=10)` → читает suggestions.jsonl, группирует похожие вопросы, возвращает топ-N по частоте

### Как будет проверяться

1. `python bot.py` запускается без ошибок (с BOT_TOKEN в .env)
2. Отправляю боту «где расписание» → получаю ответ + кнопки Да/Нет
3. Нажимаю «Да» → в requests.jsonl появляется запись с `"helpful": true`
4. Отправляю «как оплатить парковку» (нет в FAQ) → бот предлагает записать вопрос
5. Соглашаюсь → в suggestions.jsonl появляется запись
6. Отправляю `/gaps` → вижу список вопросов без ответа
7. faq.json содержит ≥ 30 записей с правильными полями по FORMATS.md
