### Что нового в Sprint 3

Сейчас аналитика Макара — это текстовый отчёт. Но FAQ-бот начнёт копить данные в `requests.jsonl` (логи запросов) и `suggestions.jsonl` (вопросы без ответа). Эти данные нужно визуализировать, чтобы понимать: что спрашивают, где бот не справляется, какие FAQ добавить.

**Новая функциональность:** интерактивный HTML-дашборд, который читает requests.jsonl и suggestions.jsonl и показывает аналитику.

### Задание

1. **Генератор тестовых данных** — скрипт `generate_test_data.py`, который создаёт реалистичные requests.jsonl (~200 записей) и suggestions.jsonl (~30 записей) для тестирования дашборда.

2. **Дашборд** — `dashboard.html` (один файл, открывается в браузере):
   - Загрузка данных: кнопка «Загрузить requests.jsonl» (через `<input type="file">`)
   - Карточки KPI: всего запросов, % с ответом, % «Помогло», среднее score
   - График: количество запросов по дням/часам
   - Топ-10 самых частых вопросов (таблица)
   - Список «пробелов» — вопросы, на которые бот не ответил (из suggestions.jsonl), сгруппированные по похожести
   - Фильтр по дате

3. **Экспорт рекомендаций** — кнопка «Экспорт в CSV»: список вопросов-кандидатов на добавление в FAQ, отсортированных по частоте.

### Результат: что лежит в папке

```
analytics/
├── dashboard.html            # Интерактивный дашборд (один файл)
├── generate_test_data.py     # Генератор тестовых данных
├── test_data/
│   ├── requests.jsonl        # Тестовые логи (~200 записей)
│   └── suggestions.jsonl     # Тестовые предложения (~30 записей)
└── README.md                 # Как запустить, скриншот дашборда
```

### Как будет проверяться

1. `python generate_test_data.py` → создаёт test_data/requests.jsonl + suggestions.jsonl
2. Открываю `dashboard.html` в браузере
3. Загружаю requests.jsonl → вижу KPI-карточки, график, топ вопросов
4. Загружаю suggestions.jsonl → вижу список пробелов
5. Нажимаю «Экспорт в CSV» → скачивается файл с рекомендациями
