# Sprint 4 — Макар Леонардов

## Интерактивный граф знаний

Визуализировать базу знаний `faq.json` как интерактивный граф на D3.js. Абитуриент или студент может исследовать: какие темы есть, как они связаны, что можно спросить у бота.

## Что нужно сделать

### Новый файл: `knowledge_graph.html`

Одностраничное приложение на D3.js (force-directed graph).

**Структура графа:**

```
Категория (большой узел)
  └── Вопрос (средний узел)
        └── Ключевое слово (маленький узел)
```

Пример:
```
[экзамены] ──→ [Когда пересдача по матану?] ──→ [пересдача]
                                              ──→ [матан]
                                              ──→ [экзамен]
```

**Функциональность:**

1. **Загрузка данных** — читает `faq.json` (File API как в дашборде, или захардкоженный путь)

2. **Визуализация** — D3.js force-directed graph:
   - Узлы-категории: большие, цветные (каждая категория — свой цвет)
   - Узлы-вопросы: средние, серые
   - **Ключевые слова не рисовать как отдельные узлы** — при 62+ записях это даст 350+ узлов и граф зависнет. Вместо этого показывать ключевые слова в карточке при клике на вопрос.
   - Рёбра: категория → вопросы

3. **Интерактивность:**
   - Клик на узел вопроса → всплывает карточка с вопросом, ответом и ключевыми словами
   - Hover → подсвечивает узел и связанные с ним вопросы категории
   - Drag → можно перетаскивать узлы
   - Фильтр по категории (чекбоксы или кнопки сверху)
   - Поиск по вопросу (input → подсвечивает найденные узлы)

4. **Легенда** — цвета категорий

**Технически:**
```javascript
// Загрузка и трансформация данных
const nodes = [];
const links = [];

faqData.forEach(entry => {
  // добавить узел категории (если нет)
  // добавить узел вопроса
  // добавить узлы ключевых слов
  // добавить рёбра
});

// D3 force simulation
const simulation = d3.forceSimulation(nodes)
  .force("link", d3.forceLink(links))
  .force("charge", d3.forceManyBody())
  .force("center", d3.forceCenter(width/2, height/2));
```

## Как проверяется

1. Открыть `knowledge_graph.html` в браузере
2. Загрузить `faq.json` → появляется граф с узлами и рёбрами
3. Клик на вопрос → видна карточка с ответом
4. Фильтр по категории → показываются только нужные узлы
5. Поиск «экзамен» → подсвечиваются связанные узлы
6. Красиво выглядит на большом экране стенда
