# Sprint 5 — Макар Леонардов

## Дашборд аналитики: финализация

В Sprint 3 ты сделал интерактивный дашборд на Chart.js. В Sprint 4 коммитов не было — это нужно исправить. Сейчас дашборд работает только на тестовых данных и при наличии интернета (Chart.js грузится с CDN). Доводим до состояния «можно показать реальные данные с ДОД».

## Задачи

### 1) Скачай Chart.js файлом

Сейчас в `dashboard.html` стоит:
```html
<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
```

Без интернета это сломается. На демо ДОД интернет может лагать. Скачай файл локально:

```bash
mkdir -p analytics/vendor
curl -o analytics/vendor/chart.min.js https://cdn.jsdelivr.net/npm/chart.js
```

В `dashboard.html` поменяй `src` на:
```html
<script src="vendor/chart.min.js"></script>
```

### 2) Совместимость с форматом бота Артёма+Льва

Дашборд парсит файлы `requests.jsonl` и `suggestions.jsonl`. Проверь что твой парсер понимает РЕАЛЬНЫЙ формат который пишет бот:

1. Подними сервер Артёма у себя (читай его README в `student_assistant/bot/`)
2. Сделай пару запросов
3. Открой получившийся `requests.jsonl` — посмотри какие там поля
4. Сверь со своим парсером, при расхождении — поправь под реальный формат

### 3) README

```markdown
# Дашборд аналитики

Показывает статистику запросов к ассистенту: KPI, графики, топ вопросов, пробелы в базе знаний.

## Запуск

Просто открой `dashboard.html` в браузере — никакого сервера не нужно.

## Какие данные загружать

В дашборде есть две кнопки загрузки:
- `requests.jsonl` — лог запросов (создаёт бот Артёма+Льва в `student_assistant/bot/`)
- `suggestions.jsonl` — лог предложенных вопросов (там же)

После загрузки появятся KPI, графики и таблица топ-вопросов.

## Генератор тестовых данных

python generate_test_data.py

Создаст файлы для тестирования дашборда без реальных логов.
```

## Как проверяется

1. Другой студент клонирует репо
2. Открывает `analytics/dashboard.html` в браузере БЕЗ интернета — Chart.js загружается из локального файла
3. Загружает в дашборд `requests.jsonl` и `suggestions.jsonl` от реального бота — графики и KPI показываются корректно
4. Запускает `python generate_test_data.py` — тестовые данные генерируются
