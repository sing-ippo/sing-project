# Sprint 4 — Павел Чернов + Спартак Петросян

## Фикс + квиз из базы знаний студентов

Два задания: обязательный фикс критических проблем из Sprint 3, и новая функция — генерация квиза прямо из faq.json.

## Часть 1 — Обязательные фиксы (до среды)

### 1. Убрать хардкод прокси

В `teacher_bot.py` сейчас:
```python
PROXY_URL = 'http://127.0.0.1:10809'
os.environ['HTTP_PROXY'] = PROXY_URL
os.environ['HTTPS_PROXY'] = PROXY_URL
```

Бот не запустится ни у кого кроме Павла. Убрать эти строки полностью.
Если прокси нужен — вынести в `.env`:
```
PROXY_URL=http://127.0.0.1:10809
```
И читать через `os.getenv("PROXY_URL")` только если задана.

### 2. Перевыпустить токен бота

Файл `.env` был запушен в git в Sprint 3 — токен скомпрометирован.
- Зайти в @BotFather → `/mybots` → выбрать бота → `API Token` → `Revoke`
- Получить новый токен
- Положить в `.env` (который в `.gitignore`)

### 3. Проверить .gitignore

Убедиться что `.env` и `downloads/` в `.gitignore`.

## Часть 2 — Новая функция: квиз из faq.json

Сейчас бот генерирует квиз только из загруженного PDF. Новая функция: преподаватель может попросить квиз по теме прямо из базы знаний студентов.

**Новая команда `/quiz_faq <категория>`**

Например: `/quiz_faq экзамены` → бот берёт записи из faq.json с категорией `экзамены` и генерирует квиз через Gemini API.

**Что сделать:**

1. Добавить в `pipeline.py` функцию `generate_quiz_from_faq(entries: list, num_questions=5)`:
   - Принимает список FAQ-записей
   - Формирует промпт: "На основе этих вопросов и ответов составь тест..."
   - Возвращает quiz в том же формате что и `generate_quiz()`

2. Добавить в `teacher_bot.py` хэндлер команды `/quiz_faq`:
   - Читает `faq.json` — файл лежит в `student_assistant/data/faq.json`. Путь указать в `.env`:
     ```
     FAQ_PATH=../../student_assistant/data/faq.json
     ```
     И читать через `os.getenv("FAQ_PATH", "../../student_assistant/data/faq.json")`
   - Фильтрует по категории если указана
   - Запускает `generate_quiz_from_faq()`
   - Запускает интерактивный квиз (уже есть в боте)

**Павел** — `pipeline.py` (функция генерации)
**Спартак** — `teacher_bot.py` (хэндлер команды)
