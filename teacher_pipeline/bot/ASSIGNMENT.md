# Sprint 3 — Павел Чернов + Спартак Петросян

## Telegram-бот преподавателя с интерактивным квизом

### Что уже есть (Sprint 1–2)

**Павел** написал `pipeline.py` (~140 строк):
- Загрузка документа → нарезка → генерация квиза через Gemini API
- Поддержка .txt (и частично .pdf)
- Проблемы Sprint 2: hardcoded API key, неполные поля в quiz.json (нет explanation, source_chunk_id), формат чанков отличался от спеки

**Спартак** написал `teacher_bot.py` (~120 строк):
- Telegram-бот на python-telegram-bot
- /start, приём текстовых сообщений
- Отправка квиза как текст
- Проблемы Sprint 2: квиз был hardcoded (не генерировался из документа), не хватало полей в формате, API key в коде

### Что нового в Sprint 3

Сейчас: Павел генерирует квиз в файл, Спартак показывает заготовленный квиз в боте — друг с другом не связаны. Преподаватель не может загрузить документ и получить квиз «в один клик».

**Новая функциональность:** Telegram-бот, в котором преподаватель загружает документ → бот проводит интерактивный квиз с кнопками → в конце показывает результат.

1. **Приём документа** — преподаватель отправляет .txt или .pdf файл в бот.

2. **Генерация квиза** — бот нарезает файл на чанки и генерирует вопросы (через API или demo-fallback).

3. **Интерактивный квиз** — бот отправляет вопросы по одному с inline-кнопками (A / B / C / D). Показывает правильный/неправильный ответ сразу.

4. **Результат** — после последнего вопроса: «Результат: 4/5 (80%)».

### Кто что делает

**Павел:**
- `pipeline.py` — бэкенд обработки:
  - Функция `process_document(file_path: str, num_questions: int = 5) -> dict` — принимает путь к файлу, возвращает `{"chunks": [...], "quiz": [...]}`
  - Нарезка: chunk_size=400, overlap=50 (можно взять chunk_module из прототипа или написать свой)
  - Квиз: через Groq API (или Gemini, или demo-fallback)
  - Формат quiz.json строго по FORMATS.md: id, question, options (4 шт), correct (0–3), explanation, source_chunk_id
  - API-ключи ТОЛЬКО через .env (не в коде!)
  - Обработка ошибок: файл пустой, файл слишком большой (> 50 КБ), неподдерживаемый формат → понятное сообщение

**Спартак:**
- `teacher_bot.py` — Telegram-бот:
  - /start — приветствие и инструкция
  - Приём файла (document handler) → сохранить во временную папку → вызвать `process_document()`
  - Если квиз сгенерирован → отправить первый вопрос с inline-кнопками A / B / C / D
  - По нажатию кнопки: показать «Правильно!» или «Неправильно. Правильный ответ: B» + explanation
  - Автоматически отправить следующий вопрос
  - После последнего вопроса: «Результат: X/Y (Z%)»
  - Хранение состояния квиза: какой вопрос текущий, сколько правильных (можно в dict по chat_id)

**Вместе:**
- Согласовать интерфейс: `process_document()` возвращает quiz в формате, который бот умеет отображать
- .env.example с обоими ключами
- Тестирование: прогнать с 2 разными документами, убедиться что квиз проходится от начала до конца

### Как будет проверяться

1. `python teacher_bot.py` запускается без ошибок (с .env)
2. Отправляю боту `lecture_short.txt` → бот пишет «Генерирую квиз...» → присылает первый вопрос с кнопками A/B/C/D
3. Нажимаю кнопку → вижу «Правильно!» или «Неправильно. Правильный ответ: ...» + объяснение
4. Автоматически приходит следующий вопрос
5. После последнего → «Результат: X/Y (Z%)»
6. Отправляю текстовое сообщение (не файл) → бот не падает, пишет инструкцию
7. В demo_screenshot.png — скриншот реального диалога
8. В git-истории коммиты от Павла И от Спартака (≥ 3 от каждого)
